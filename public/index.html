    Welcome to the A-Bucketlist API documentation. This API exposes various endpoints which allows for creating, retrieving, updating and deleting bucketlists.

    On the right is an example terminal containing various code examples that show you how to better use the API.

    On the left are navigation helpers to help you search this document or jump through to specific sections.

    • The base url for the API is https://a-bucketlist.herokuapp.com
    • Current version of API is version 1

    Creating an account

    Request format

    # The backward slashes help to write 
# the command on multiple lines

curl -X POST -d \ 
"username=testuser&password=s3cur3p@ssw0rd!" \
"https://a-bucketlist.herokuapp.com/user/new"

    The above command returns a json response like this:

      {
    "user": {
      "id": 1,
      "username": "testuser",
      "created_on": "Dec 3, 2015"
    }
  }
    In order to gain full access to all available API endpoints, you need to register for a user account.

    You create an account by making a post request to the /user/new endpoint and passing in valid parameters as part of the request.

    The parameters to be passed in as part of the url request are:

    username: Any alphanumeric character between 2 and 50 characters.

    password: Any word or non word character between 8 and 255 characters.

    HTTP Request

    POST https://a-bucketlist.herokuapp.com/user/new

    Logging in

    Request format

    # The backward slashes help to write 
# the command on multiple lines

curl -i -X POST -d \ 
"username=testuser&password=s3cur3p@ssw0rd!" \
"https://a-bucketlist.herokuapp.com/auth/login"

    The above command returns a json response like this:

      {
    "message": "Logged in",
    "information": {
      "user": "testuser",
      "token": "eyJ0eXAiOiJ9.eyJ1c2VyX2l9.SkvNknvQlQA",
      "expires_on": "Dec 4, 2015. 5:29 am"
    }
  }

    Note: The above token is just for example purposes and is not actually a valid token

    You log in to your account by making a post request to the /auth/login endpoint and passing in valid parameters as part of the request.

    Upon successful login, an api token is generated for your account and this token has to be passed (using headers) along with every request when you want to make bucketlist operations.

    HTTP Request

    POST https://a-bucketlist.herokuapp.com/auth/login

    Logging out

    Request format

    # The backward slashes help to write 
# the command on multiple lines

curl -i -X GET -H \
"Authorization: Token token=entertokenhere" \
"https://a-bucketlist.herokuapp.com/auth/login"

    The above command returns a json response like this:

      {
    "user": "testuser",
    "message": "Successfully logged out"
  }

    You log out of your account by making a get request to the /auth/logout endpoint and passing in your api token in the authorization header.

    HTTP Request

    GET https://a-bucketlist.herokuapp.com/auth/logout

    Authentication

    To authorize, use this code:

    # With curl, you can just pass 
# the correct header with each request

curl "api_endpoint_here"
  -H "Authorization: Token token=entertokenhere"

    Make sure to replace entertokenhere with your API key.

    A-Bucketlist uses tokens for authentication to the various endpoints for creating, updating, retrieving and deleting bucketlists.

    Upon successful login, a unique token is generated and assigned to your account and it is this token that you pass along in the header when querying the API for bucketlist operations.

    A-Bucketlist expects the API token to be included in all API requests to the server in a header that looks like the following:

    Authorization: Token token=entertokenhere

    You must replace entertokenhere with your API token.
    All generated tokens have a lifetime of 12 hours after which they expire and become invalid. To request for a new one, just make a new request to the `/auth/login` endpoint passing in valid parameters as part of the request.
    Note: If you don’t pass in the token as specified, you won’t be able to gain access to the API endpoints for bucketlist operations even if you are logged in.

    Creating a bucketlist

    Request format: Creating a bucketlist

    # The backward slashes help to write 
# the command on multiple lines

curl -X POST -d "name=Bucket1" -H \
"Authorization: Token token=entertokenhere" \
"https://a-bucketlist.herokuapp.com/api/v1/bucketlists"

    The above command returns a json response like this:

      {
    "bucketlist": {
      "id": 30,
      "name": "Bucket1",
      "created_by": "1",
      "date_created": "Dec  3, 2015.  6:29 pm",
      "last_updated_on": "Dec  3, 2015.  6:29 pm",
      "items": []
    }
  }
    Accessing this API endpoint requires an authorized user with valid API token.

    A new bucketlist is created by making a post request to the /api/v1/bucketlists endpoint and passing in valid parameters as part of the request.

    HTTP Request

    POST https://a-bucketlist.herokuapp.com/api/v1/bucketlists

    Query parameter(s) for creating a bucketlist

    Parameter Optional Allowed Characters Description
    name false Alphanumeric + space character The name of the bucketlist

    Updating bucketlists

    Update a bucketlist

    # The backward slashes help to write 
# the command on multiple lines

curl -X PUT -d "name=new bucket" -H \
"Authorization: Token token=entertokenhere" \
"https://a-bucketlist.herokuapp.com/api/v1/bucketlists/28"

    The above command returns JSON structured like this:

      {
  "bucketlist": {
    "id": 28,
    "name": "New bucket",
    "created_by": "6",
    "date_created": "Dec  3, 2015.  6:27 pm",
    "last_updated_on": "Dec  4, 2015.  5:27 am",
    "items": []
  }
}
    Accessing this API endpoint requires an authorized user with valid API token.

    You update a bucketlist by making a put request to the /api/v1/bucketlists/<id> endpoint and passing in valid parameters as part of the request.

    HTTP Request

    PUT https://a-bucketlist.herokuapp.com/api/v1/bucketlists/9

    Query parameter(s) for updating a bucketlist

    Parameter Optional Allowed Characters Description
    name false Alphanumeric + space character The new name of the bucketlist

    Update a single bucketlist item

    # The backward slashes help to write 
# the command on multiple lines

curl -X PUT -d "name=New item&done=true" -H \
"Authorization: Token token=entertokenhere" \
"https://a-bucketlist.herokuapp.com/api/v1/bucketlists/28/items/17"

    The above command returns JSON structured like this:

      {
      "item": {
        "id": 17,
        "name": "New item",
        "done": true,
        "date_created": "Dec  4, 2015.  5:30 am",
        "last_updated_on": "Dec  4, 2015.  5:30 am"
      }
  }
    Accessing this API endpoint requires an authorized user with valid API token.

    This endpoint is used to update a specific bucketlist item created by a particular user. A bucketlist item can be updated by making a put request to the /api/v1/bucketlists/<id>/items/<item_id> endpoint.

    HTTP Request

    PUT https://a-bucketlist.herokuapp.com/api/v1/bucketlists/9/items/1

    Query parameter(s) for updating a bucketlist item

    Parameter Optional Allowed Characters Description
    name true Alphanumeric + space character The new name of the bucketlist
    done true t or f or long form of true or false The status of the item. True for when the item has been done and false for when it is not yet done.
    You don’t have to pass in both parameters when making a request to this endpoint. If you need to update only the name, just pass in only the name as part of the request and the same statement holds true if you need to update only the done status. If no new information is passed along with the request, then no changes are made.
    Note: Passing invalid parameters to either ‘name’ or 'done’ causes the API to make no change to the bucketlist item.

"Authorization: Token token=entertokenhere" \

curl -X GET -d "page=1&limit=2" -H \

        "id": 28,
        "name": "Things to do",
        "created_by": "6",
        "date_created": "Dec  3, 2015.  6:27 pm",

    Accessing this API endpoint requires an authorized user with valid API token.

    This endpoint is used to request for a paginated response. The default amount of result returned per page is 20 and no greater than 100 records can be returned per page for any request.

    HTTP Request

    GET https://a-bucketlist.herokuapp.com/api/v1/bucketlists

    Meta information

    The meta information displays helpful information about the paginated response. The table below lists the values and their meanings:

    Name Description
    current_page Shows the current page number
    next Shows the page number for the immediate next page of results
    total_pages Shows the total amount of pages based on the page and limit parameters
    total_count Shows the total amount of records returned
    Note: You can paginate a search result by adding the page and limit parameters as part of the search query.

    Endpoints

    All exposed endpoints are summarised in the table below:
    Action Endpoint Description Public Access
    POST /user/new Creates a user account True
    POST /auth/login Logs a user into the user’s account True
    GET /auth/logout Logs a user out False
    POST /bucketlists Creates a new bucketlist False
    GET /bucketlists Get all bucketlists False
    GET /bucketlists/ Get a single bucketlist False
    PUT /bucketlists/ Update bucketlist with the specified id False
    DELETE /bucketlists/ Delete bucketlist with the specified id False
    POST /bucketlists//items Adds a new item to the bucketlist with the specified id False
    PUT /bucketlists//items/ Update a single bucket list item that matches specified ids False
    DELETE /bucketlists//items/ Delete a single bucket list item that matches specified ids False
